Controllo Gestione

Architettura proposta:

• Backend: GET/PUT /settings/dashboard-config/{role} — salva un JSON con widget abilitati, ordine, layout e azioni rapide

• Frontend: Home.jsx legge la config per il ruolo corrente e renderizza i widget nell'ordine scelto

• Editor: nuova sezione in Impostazioni Sistema per configurare le dashboard

• Default: ogni ruolo ha una config di default se non personalizzata

• Widget = componenti indipendenti — ognuno sa fetchare i propri dati e renderizzarsi
